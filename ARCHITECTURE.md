# توثيق البنية التقنية — منصة فلاش

## 1) قاعدة البيانات (الجداول الرئيسية)
- `users` — كل الحسابات (زبون/سائق/صاحب مطعم/أدمن)، تتضمن `reliability_score` (تقييم أمانة، ميزة خاصة بالسوق الليبي لتقليل الطلبات الوهمية) و`last_login_ip`.
- `wallets` — محفظة واحدة لكل مستخدم، بعملة قابلة للتوسع (افتراضي LYD)، وحقل `is_frozen` للتجميد اليدوي.
- `transactions` — سجل كامل (ledger) لكل حركة مالية: شحن، تحويل صادر/وارد، دفع طلب، تعديل إداري.
- `payment_method_blocks` — حظر وسيلة دفع محددة (نقدي/إلكتروني/محفظة) لحساب محدد، مع السبب والمسؤول.
- `bans` — حظر بحسب النوع: `phone` أو `ip` أو `user_id`، قابل لتحديد تاريخ انتهاء أو حظر دائم.
- `feature_flags` — كل ميزة في المنصة لها مفتاح `key` فريد + `is_enabled` يتحكم به الأدمن مباشرة.
- `banners` — بانرات الصفحة الرئيسية مع فترة عرض وترتيب.
- `offers` — عروض/إعلانات صفحة العروض المستقلة.
- `restaurants` — بيانات المطعم + `is_open` (تحكم صاحب المطعم) + `is_approved` (تحكم الأدمن).
- `orders` — الطلب الكامل بحالاته المتسلسلة.
- `driver_locations` — آخر موقع معروف لكل سائق + حالة الاتصال، تُغذّي الخريطة الحرارية والتتبع اللحظي.

## 2) مبدأ "التحكم اليدوي بكل شيء"
كل مسار حسّاس في الـ API محاط بـ middleware يُدعى `featureFlagGuard('feature.key')`.
مثال من `walletRoutes.js`:
```js
router.post('/transfer', featureFlagGuard('wallet.p2p_transfer'), ctrl.transfer);
```
إن عطّل الأدمن هذا المفتاح من لوحة التحكم، سيحصل أي طلب على `423 Locked` فورًا. لإضافة ميزة جديدة قابلة للتحكم:
1. أضف سطرًا في `backend/src/config/featureFlags.js`.
2. شغّل `npm run seed` لإنشائه في قاعدة البيانات.
3. لُفّ الـ route المعني بـ `featureFlagGuard('key')`.
4. سيظهر تلقائيًا في لوحة تحكم الأدمن ضمن صفحة "التحكم بالميزات".

## 3) تدفق الدفع
`paymentService.processOrderPayment()` هو المسار الموحّد لكل عملية دفع:
1. يتحقق أن الوسيلة (نقدي/إلكتروني) مفعّلة عمومًا عبر `feature_flags`.
2. يتحقق أن الوسيلة غير محظورة يدويًا على هذا الحساب تحديدًا عبر `payment_method_blocks`.
3. ينفّذ العملية (خصم من المحفظة، أو تسجيل الدفع النقدي عند الاستلام، أو تحويل لبوابة إلكترونية).

## 4) الحظر (Bans)
`checkBan` middleware يعمل على `login` و`register` قبل أي تحقق آخر، ويفحص:
- رقم الهاتف المُرسَل في الطلب مقابل جدول `bans` (type=phone).
- عنوان IP الحقيقي للطلب (`ipHelper.getClientIp`) مقابل (type=ip).
كل فحص له مفتاح تفعيل مستقل (`security.phone_ban_check`, `security.ip_ban_check`) يمكن إيقافه مؤقتًا عند الحاجة.

## 5) الخريطة الحرارية
`heatmapService.buildHeatmap()` يقسّم المنطقة الجغرافية إلى شبكة خلايا (حجم الخلية قابل للتعديل عبر `CELL_SIZE_DEG`)، ويحسب لكل خلية:
- `demand`: عدد الطلبات في آخر N ساعة (قابلة للتحديد عبر `?since_hours=`).
- `supply`: عدد السائقين المتصلين حاليًا في نفس الخلية.
- `intensity`: نسبة الطلب إلى العرض — هذا ما يُلوَّن على الخريطة في تطبيق فلاش مان.

## 6) قابلية التوسع
- **متعدد المدن**: عمود `city_id` موجود في `users` و`restaurants` جاهز لتفعيل الفرز الجغرافي دون تعديل بنيوي.
- **متعدد العملات**: `wallets.currency` مستقل لكل محفظة.
- **فصل الخدمات مستقبلًا**: بنية `controllers/services/models` تسمح باستخراج أي خدمة (المحفظة، الطلبات، الخريطة الحرارية) إلى microservice مستقل لاحقًا دون إعادة كتابة المنطق.
- **الكاش**: مفاتيح الميزات مخزّنة في كاش داخل الذاكرة (TTL=30 ثانية) لتقليل الضغط على قاعدة البيانات مع بقاء التغييرات شبه فورية؛ يمكن استبدالها بـ Redis عند الحاجة لتوسع أفقي (multi-instance).

## 7) نقاط النهاية (API) — ملخص
| المسار | الوصف |
|---|---|
| `POST /api/auth/register` | تسجيل حساب جديد (يُفحص بالحظر + مفتاح `orders.new_registrations`) |
| `POST /api/auth/login` | تسجيل الدخول |
| `GET /api/wallet/me` | رصيد المحفظة |
| `POST /api/wallet/topup` | شحن المحفظة |
| `POST /api/wallet/transfer` | تحويل رصيد لزبون آخر |
| `POST /api/orders` | إنشاء طلب جديد + معالجة الدفع |
| `PATCH /api/orders/:id/status` | تحديث حالة الطلب |
| `POST /api/driver/online` / `offline` | تفعيل/إيقاف اتصال السائق |
| `GET /api/driver/heatmap` | بيانات الخريطة الحرارية |
| `GET /api/public/banners` | بانر الصفحة الرئيسية |
| `GET /api/public/offers` | صفحة العروض |
| `GET/POST /api/admin/feature-flags` | التحكم بكل الميزات |
| `GET/POST /api/admin/bans` | إدارة الحظر |
| `GET/POST /api/admin/payment-blocks` | حظر وسيلة دفع لحساب |
| `GET /api/admin/users` | إدارة الحسابات |
