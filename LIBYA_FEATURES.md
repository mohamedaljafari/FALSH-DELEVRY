# ميزات مقترحة لملاءمة السوق الليبي

| الميزة | المشكلة التي تحلّها | الحالة |
|---|---|---|
| **تقييم الأمانة (Reliability Score)** | تقليل الطلبات الوهمية/الملغاة بسبب ضعف الثقة الرقمية | مُنفّذة (`users.reliability_score`) |
| **الدفع النقدي كخيار أساسي دائمًا متاح** | ضعف انتشار البطاقات البنكية والدفع الإلكتروني | مُنفّذة، وقابلة للتعطيل الكلي أو الجزئي (لحساب معيّن) عند الحاجة |
| **حظر وسيلة دفع لحساب معيّن** | التعامل مع حالات إساءة استخدام الدفع النقدي (رفض الاستلام المتكرر) دون حظر الحساب كاملًا | مُنفّذة |
| **وضع رمضان/الأعياد** | تغيّر أوقات الذروة والطلب بشكل كبير في المناسبات | مفتاح جاهز (`libya.ramadan_mode`) يحتاج ربطًا بواجهة العروض عند التفعيل |
| **تأكيد الطلب عبر SMS عند ضعف الشبكة** | تذبذب شبكات الإنترنت في بعض المناطق/الأوقات | مفتاح جاهز (`libya.offline_confirmation`)، يحتاج تكاملًا مع مزوّد SMS محلي |
| **رسوم توصيل ديناميكية** | تفاوت تكلفة الوقود والمسافات بين المدن والأحياء | مفتاح جاهز (`orders.dynamic_delivery_fee`)، الحساب الحالي بسيط ويمكن تطويره لاحقًا بخوارزمية مسافة/ازدحام |
| **نقاط الولاء** | تحفيز التكرار في سوق حساس للسعر | مفتاح جاهز (`marketing.loyalty_points`)، الجداول غير مُنفّذة بعد ويمكن إضافتها كموديول منفصل |
| **دعم دينار ليبي (LYD) افتراضيًا** | العملة المحلية الأساسية | مُنفّذة، مع بنية تسمح بعملات أخرى مستقبلًا |
| **عمولة مطعم قابلة للتخصيص لكل مطعم** | تفاوت القدرة التفاوضية بين المطاعم الكبيرة والصغيرة | مُنفّذة (`restaurants.commission_rate`) |
| **موافقة الأدمن على المطاعم قبل الظهور** | ضبط الجودة في سوق ناشئ | مُنفّذة (`restaurants.is_approved`) |

> جميع الميزات أعلاه تلتزم بمبدأ "التحكم اليدوي": أي ميزة جديدة تُضاف كمفتاح في `feature_flags` أولًا، ثم تُفعَّل تدريجيًا بعد الاختبار.
